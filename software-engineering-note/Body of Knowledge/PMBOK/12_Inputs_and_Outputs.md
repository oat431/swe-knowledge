# 12 — Inputs and Outputs

> *Source: PMBOK® Guide — Eighth Edition by PMI, §4 Inputs and Outputs (pp. 113–144)*

## Purpose

This section provides a comprehensive alphabetical catalog of all inputs and outputs referenced across the 40 project management processes within the seven performance domains. Each entry includes a brief description and identifies which processes consume the input or produce the output. Use this as a quick-reference when tracing information flow between processes.

## Inputs & Outputs Catalog

### Activity Attributes
Multiple attributes associated with each schedule activity (ID, WBS ID, labels, descriptions, predecessors, successors, logical relationships, leads/lags, resource requirements, imposed dates, constraints, assumptions). Used for schedule development and report generation. **Processes:** Develop Schedule (I), Monitor and Control Schedule (I).

### Activity List
A documented tabulation of schedule activities showing activity description, identifier, and scope of work description. **Processes:** Develop Schedule (I), Monitor and Control Schedule (I).

### Agreements
Any document or communication defining initial intentions of a project — contracts, SLAs, MOUs, letters of agreement, purchase orders, emails. **Processes:** Plan Sourcing Strategy (O), Acquire Resources (I).

### Approved Change Requests
Change requests processed through the change management plan and approved, deferred, or rejected. Implemented via Manage Project Execution. **Processes:** Assess and Implement Changes (O), Manage Project Execution (I).

### Approved Changes
Changes or modifications being or already implemented as part of project scope, resulting from fully approved change requests. **Processes:** Monitor and Control Project Performance (I).

### Assumption Log
A project document recording all assumptions and constraints throughout the project life cycle. **Processes:** Initiate Project or Phase (O), various planning processes (I).

### Backlog
An ordered list of work to be done (often user stories), prioritized to manage adaptive/agile project work. Types: product backlog, iteration backlog, release backlog. **Processes:** Plan Scope Management (O), Develop Scope Structure (O).

### Basis of Estimates
Supporting documentation detailing how estimates were derived — assumptions, constraints, confidence levels, and data sources. **Processes:** Estimate Costs (O), Estimate Resources (O), Develop Schedule (O).

### Benefits Management Plan
Documents the planned benefits of the project and how they will be measured and sustained. **Processes:** Initiate Project or Phase (O).

### Business Case
A documented economic feasibility study establishing the validity of benefits to justify the project investment. **Processes:** Initiate Project or Phase (I).

### Business Documents
Collective term for the business case and benefits management plan. **Processes:** Initiate Project or Phase (I).

### Change Log
A comprehensive list of all change requests submitted during the project, including their status (approved, deferred, rejected). **Processes:** Assess and Implement Changes (O), Monitor and Control Project Performance (I).

### Change Management Plan
Describes how change requests will be managed, documented, approved, or rejected throughout the project. **Processes:** Integrate and Align Project Plans (O).

### Change Requests
A formal proposal to modify any document, deliverable, or baseline. **Processes:** Monitor and Control Project Performance (O), various control processes.

### Communications Management Plan
Describes how project communications will be planned, structured, implemented, and monitored. **Processes:** Plan Communications Management (O).

### Configuration Management Plan
Describes how configuration items will be identified, controlled, and verified. **Processes:** Integrate and Align Project Plans (O).

### Cost Baseline
The approved version of the time-phased project budget, excluding management reserves. **Processes:** Develop Budget (O), Monitor and Control Finances (I).

### Cost Estimates
Quantitative assessments of the likely costs for resources required to complete project activities. **Processes:** Estimate Costs (O), Develop Budget (I).

### Cost Forecasts
Predictions of future project costs based on performance to date and expected future conditions. **Processes:** Monitor and Control Finances (O).

### Development Approach
The method used to create and evolve the project deliverable (predictive, adaptive, hybrid). **Processes:** Integrate and Align Project Plans (I).

### Duration Estimates
Quantitative assessments of the likely number of work periods needed to complete activities. **Processes:** Develop Schedule (I).

### Enterprise Environmental Factor Updates
Changes to EEFs identified during project execution that may affect future processes. **Processes:** Various executing and monitoring processes (O).

### Final Report
A document summarizing project performance, including achievements against objectives, variances, and lessons learned. **Processes:** Close Project or Phase (O).

### Financial Management Plan
Describes how project costs will be planned, structured, estimated, budgeted, and controlled. **Processes:** Plan Financial Management (O).

### Funding Proposals
Requests for funding to support the project budget. **Processes:** Develop Budget (O).

### Funding Strategy
The approach for securing project funding — internal allocation, external investment, grants, loans. **Processes:** Plan Financial Management (O).

### Issue Log
A project document recording all issues, their status, assigned owners, and resolution. **Processes:** Manage Project Execution (O), Monitor and Control Project Performance (I).

### Lessons Learned Register
A repository of knowledge gained during the project, used to improve future performance. **Processes:** Manage Project Knowledge (O), various processes (I).

### Milestone List
A list of significant points or events in the project with zero duration — used for tracking major deliverables. **Processes:** Develop Schedule (O).

### Organizational Process Asset Updates
Changes to OPAs identified during the project. **Processes:** Various processes (O).

### Physical or Virtual Resource Assignments
Documentation of which physical or virtual resources are assigned to which activities. **Processes:** Acquire Resources (O).

### Procurement Documentation
All documents used in procurement — RFPs, bids, contracts, statements of work. **Processes:** Plan Sourcing Strategy (O).

### Procurement Management Plan
Describes how procurement processes will be managed from sourcing strategy through contract closure. **Processes:** Plan Sourcing Strategy (O).

### Project Calendars
Calendars identifying working days, shifts, and nonworking periods for project activities. **Processes:** Develop Schedule (I).

### Project Charter
A document formally authorizing the project and providing the project manager authority to apply resources. **Processes:** Initiate Project or Phase (O).

### Project Communications
All communications artifacts produced during the project — presentations, reports, emails, meeting minutes. **Processes:** Manage Communications (O).

### Project Document Updates
Changes to any project document resulting from process execution. **Processes:** Most processes (O).

### Project Funding Requirements
Total funding requirements and periodic funding needs derived from the cost baseline. **Processes:** Develop Budget (O).

### Project Management Plan
The document describing how the project will be executed, monitored, controlled, and closed. Consolidates all subsidiary management plans. **Processes:** Integrate and Align Project Plans (O), most processes (I).

### Project Schedule
The output of schedule development — presents linked activities with planned dates, durations, milestones, and resources. **Processes:** Develop Schedule (O), Monitor and Control Schedule (I).

### Project Schedule Network Diagram
A graphical representation of logical relationships among project schedule activities. **Processes:** Develop Schedule (O).

### Project Scope Statement
A detailed description of project scope, deliverables, assumptions, constraints, and acceptance criteria. **Processes:** Define Scope (O).

### Project Team Assignments
Documentation of which team members are assigned to which roles and activities. **Processes:** Acquire Resources (O), Lead the Team (I).

### Quality Control Measurements
Results of quality control activities used to analyze and evaluate quality. **Processes:** Manage Quality Assurance (I).

### Quality Management Plan
Describes how quality policies will be implemented — quality standards, objectives, roles, and responsibilities. **Processes:** Integrate and Align Project Plans (O).

### Quality Metrics
Specific measurements used to assess whether deliverables meet quality requirements. **Processes:** Manage Quality Assurance (O).

### Quality Reports
Documents summarizing quality assurance findings, recommendations, and corrective actions. **Processes:** Manage Quality Assurance (O).

### Quality Test and Evaluation Documents
Documents specifying testing approaches, test cases, and evaluation criteria for deliverables. **Processes:** Manage Quality Assurance (I).

### Requirements Documentation
A record of all stakeholder, solution, and project requirements with their attributes. **Processes:** Elicit and Analyze Requirements (O).

### Requirements Management Plan
Describes how requirements will be elicited, analyzed, documented, and managed. **Processes:** Plan Scope Management (O).

### Requirements Traceability Matrix
A grid linking requirements from origin through to the deliverables that satisfy them. **Processes:** Elicit and Analyze Requirements (O).

### Resource Breakdown Structure (RBS)
A hierarchical representation of resources by category and type. **Processes:** Estimate Resources (O).

### Resource Calendar
Identifies the working days, shifts, and availability of specific resources. **Processes:** Acquire Resources (I), Develop Schedule (I).

### Resource Management Plan
Describes how project resources will be categorized, allocated, managed, and released. **Processes:** Plan Resource Management (O).

### Resource Requirements
Identifies the types and quantities of resources required for each work package or activity. **Processes:** Estimate Resources (O), Acquire Resources (I).

### Revenue Forecasts
Predictions of future project revenue based on performance and market conditions. **Processes:** Monitor and Control Finances (O).

### Risk Management Plan
Describes how risk management activities will be structured and performed. **Processes:** Plan Risk Management (O).

### Risk Register
A repository for identified risks with their analysis, responses, and current status. **Processes:** Identify Risks (O), most risk processes (I/O).

### Risk Report
Summarizes individual and overall project risks for senior management review. **Processes:** Perform Risk Analysis (O).

### Schedule Baseline
The approved version of the schedule model, used as a basis for comparison to actual results. **Processes:** Develop Schedule (O), Monitor and Control Schedule (I).

### Schedule Data
The collection of information supporting the schedule — activity attributes, assumptions, constraints. **Processes:** Develop Schedule (O).

### Schedule Forecasts
Predictions of future schedule conditions based on performance to date. **Processes:** Monitor and Control Schedule (O).

### Schedule Management Plan
Describes how the schedule will be developed, monitored, and controlled. **Processes:** Plan Schedule Management (O).

### Scope Baseline
The approved version of the scope statement, WBS, and WBS dictionary — used for comparison. **Processes:** Develop Scope Structure (O), Monitor and Control Scope (I).

### Scope Management Plan
Describes how scope will be defined, developed, monitored, controlled, and validated. **Processes:** Plan Scope Management (O).

### Skill Matrix
A mapping of team members' skills, competencies, and experience levels against project needs. **Processes:** Estimate Resources (I), Acquire Resources (I).

### Sourcing Strategy Plan
Documents the approach for acquiring goods and services — make-or-buy decisions, contract types, supplier selection. **Processes:** Plan Sourcing Strategy (O).

### Stakeholder Engagement Plan
Describes strategies and actions to engage stakeholders based on their needs, interests, and impact. **Processes:** Plan Stakeholder Engagement (O).

### Stakeholder Register
A repository of identified stakeholders with their assessment information, classification, and engagement needs. **Processes:** Identify Stakeholders (O).

### Status Report
A periodic summary of project status — progress against baselines, achievements, issues, risks. **Processes:** Monitor and Control Project Performance (O).

### Team Charter
Documents team values, agreements, operating guidelines, and behavioral expectations. **Processes:** Plan Resource Management (O).

### Team Performance Assessments
Evaluations of team performance against goals — skill improvements, turnover, team cohesion. **Processes:** Lead the Team (O).

### User Stories
Brief descriptions of a feature from an end-user perspective, capturing the who, what, and why. **Processes:** Elicit and Analyze Requirements (O).

### Verified Deliverables
Deliverables that have been completed and checked for correctness through quality control. **Processes:** Monitor and Control Scope (I), Validate Scope (I).

### Work Performance Data
Raw observations and measurements identified during activities performed to carry out project work. **Processes:** Manage Project Execution (O), various monitoring processes (I).

### Work Performance Information
Performance data integrated and analyzed in context — provides status against baselines. **Processes:** Various monitoring and controlling processes (O).

### Work Performance Reports
Physical or electronic representations of work performance information — status reports, dashboards, memos. **Processes:** Monitor and Control Project Performance (O).
