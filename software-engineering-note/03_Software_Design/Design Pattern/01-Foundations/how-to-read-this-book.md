> *Source: Dive Into Design Patterns by Alexander Shvets, "How to Read This Book" (p. 6)*

> The book describes 22 classic GoF design patterns. You can read cover-to-cover or jump to patterns of interest via cross-reference anchors. All code is presented in pseudocode to remain language-agnostic. Readers new to OOP should brush up on key OOP terms and UML diagram basics first.

This chapter serves as a navigation guide for the book. It establishes two reading paths—sequential or interest-driven—and explains the connective tissue between patterns through cross-references. It also notes that pseudocode keeps the material universal and suggests the OOP primer for those who need it.

## Key Points

- **22 GoF patterns** — The book covers all 22 classic patterns formulated by the Gang of Four in 1994
- **Flexible reading order** — Read cover-to-cover or pick individual patterns by interest; each chapter is self-contained
- **Cross-reference anchors** — Related patterns are linked at the end of each chapter, allowing easy topic jumping
- **Pseudocode only** — All code samples are language-agnostic pseudocode, so the material applies universally
- **OOP refresh recommended** — Before diving into patterns, readers can review key OOP terminology and UML diagram basics in the preceding chapter
- **Skip if confident** — If you already know OOP and UML, jump straight into pattern study
