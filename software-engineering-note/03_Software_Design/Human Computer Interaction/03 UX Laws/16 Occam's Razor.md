# 16 Occam's Razor

> **The simplest solution is usually the best.**

Among competing designs that achieve the same goal, choose the one with the fewest elements, steps, or assumptions. Every extra element adds cognitive load and potential confusion.

---

## The Principle

![Occam's Razor - simplicity wins](https://lawsofux.com/images/occams-razor.svg)

*Source: lawsofux.com*

---

## In UI Design

| ❌ Over-Designed | ✅ Simplified |
|-----------------|--------------|
| Homepage with carousel, 3 hero CTAs, animated background, chat widget, newsletter popup | One headline, one CTA, one supporting image |
| 8-column pricing table with feature comparison matrix | 3 tiers: Basic, Pro, Enterprise — with a "Most Popular" badge |
| Settings page with 50 individual toggles | Grouped categories with smart defaults |

---

## How to Apply It

Before adding anything to your UI, ask:

| Question | If "No" → Cut It |
|----------|-----------------|
| Does this help the user complete their task? | Cut it |
| Could the user achieve the same goal without this? | Cut it |
| Is this the simplest way to communicate this? | Simplify it |
| Would removing this break anything? | Cut it |

---

## The "Cut It in Half" Rule

When you think your design is done, try cutting the number of:

- Words in headlines (half them)
- Fields in your form (remove optional ones)
- Items in navigation (move to footer/secondary)
- Buttons on the page (one primary, rest are links)

---

## Occam's Razor ≠ Minimalism

| Minimalism (Style) | Occam's Razor (Principle) |
|-------------------|--------------------------|
| A design aesthetic — few elements | A decision principle — simplest SOLUTION |
| Can go too far (removing necessary info) | Doesn't remove what's necessary — removes what's unnecessary |

> "A design is finished not when there's nothing more to add, but when there's nothing left to take away." — Antoine de Saint-Exupéry
