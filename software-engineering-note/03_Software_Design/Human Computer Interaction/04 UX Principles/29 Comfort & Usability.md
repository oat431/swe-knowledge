# 29 Comfort & Usability

> **If using your product feels like work, users will find one that doesn't.**

Comfort in UX means: low cognitive load, minimal physical effort, emotional ease. Users shouldn't have to think hard, click precisely, or feel anxious to use your product.

---

## Cognitive Comfort

Reduce mental effort:

| Rule | Example |
|------|---------|
| **Recognition over recall** | Show options — don't make users remember them. Autocomplete > free text. |
| **Chunk information** | Break forms into steps. Group settings into categories. |
| **Clear language** | "Save changes" not "Persist modifications to data store" |
| **Smart defaults** | Pre-fill country, select most common option, default to "today's date" |

---

## Physical Comfort

Reduce physical effort:

| Rule | Example |
|------|---------|
| **Minimize clicks/taps** | Auto-advance fields (entering 4-digit code auto-submits). One-click checkout. |
| **Thumb-friendly mobile** | Primary actions in the bottom half of the screen — reachable with one hand |
| **Reduce scrolling** | Put key info above the fold. Sticky CTAs for long pages. |
| **Dark mode** | Reduces eye strain at night |

---

## Emotional Comfort

Reduce anxiety and friction:

| ❌ Anxiety-Inducing | ✅ Comforting |
|--------------------|--------------|
| "Are you sure you want to delete? This action cannot be undone." (scary) | Deleted items go to Trash. "Undo" button appears. |
| "Error 500: Internal Server Error" | "Something went wrong. We're on it. Your data is safe." |
| Hidden prices until checkout | Transparent pricing from the start |
| "Your session has expired. Log in again." | Auto-save drafts. Graceful session recovery. |

---

## The "Don't Make Me Think" Principle

Steve Krug's first law of usability:

> If a user has to stop and think about how something works — even for a second — the design has failed.

| Making Users Think | Not Making Them Think |
|-------------------|----------------------|
| "Is this clickable?" | Clear visual affordance (button looks like a button) |
| "Where am I in this process?" | Progress indicator, breadcrumbs |
| "What happens if I click this?" | Clear labels: "Save and Continue" not "Submit" |
| "Did that work?" | Instant feedback: success message, animation, next step |
