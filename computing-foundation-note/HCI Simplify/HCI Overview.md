# Human-Computer Interaction (Simplified)

HCI is the study of how humans interact with computers and how to design interfaces that work with human psychology, not against it. Every developer builds interfaces — whether it's a web UI, an API, a CLI, or an error message. Understanding HCI basics means fewer frustrated users.

---

## Why Developers Need This

| Without HCI knowledge | With HCI knowledge |
|---|---|
| "The user is stupid" | "The interface failed the user" |
| Features nobody can find | Discoverable, intuitive flows |
| Error messages that confuse | Error messages that guide |
| Accessibility as afterthought | Accessibility built in |

---

## Core Concepts

| Concept | What It Means |
|---------|---------------|
| **Usability** | Can users accomplish their goals effectively, efficiently, and satisfactorily? |
| **Cognitive Load** | How much mental effort the interface demands. Less = better. |
| **Mental Models** | Users have expectations based on past experience. Match them or explicitly teach. |
| **Affordance** | A button looks clickable. A handle looks pullable. Design should signal its use. |
| **Feedback** | Every action needs a visible response. No feedback = "did it work?" anxiety. |
| **Consistency** | Same action, same result, everywhere. Internal + external consistency. |
| **Accessibility** | Designing for ALL users, including those with disabilities. Not optional. |
