# Machine Learning (Ch 18-20)

## 1. Forms of Learning

Learning depends on three factors: **improved components**, **available feedback**, and **representation used**.

### Three Types of Feedback

| Type | Description | Example |
|---|---|---|
| **Supervised Learning** | Learn from input–output pairs | Recognizing buses from photos |
| **Unsupervised Learning** | Discover patterns in unlabeled data | Clustering "good/bad traffic days" |
| **Reinforcement Learning** | Learn from reward/punishment sequences | Tips → driving quality |

> Semi-supervised learning lies between supervised and unsupervised, leveraging a small amount of labeled data + a large amount of unlabeled data.

---

## 2. Supervised Learning Framework

**Task:** Given training set $\{(x_j, y_j)\}_{j=1}^N$, where $y = f(x)$ is generated by an unknown function $f$, find a hypothesis $h$ that approximates $f$.

- **Classification:** Output $y$ is discrete
- **Regression:** Output $y$ is continuous
- **Hypothesis Space $H$:** Set of candidate hypotheses
- **Generalization:** Prediction ability on unseen data

### Ockham's Razor

Choose the **simplest** hypothesis consistent with the data.

---

## 3. Decision Trees

### Representation
- Tree structure: internal nodes = attribute tests, branches = attribute values, leaves = classification results
- Can represent all Boolean functions (but exponential size)

### Learning Algorithm (ID3/C4.5)

Recursively select the attribute with maximum **information gain** to split:

$$\text{Gain}(A) = H(\text{examples}) - \sum_{v \in \text{Values}(A)} \frac{|\text{examples}_v|}{|\text{examples}|} H(\text{examples}_v)$$

Where $H$ is entropy.

### Overfitting and Pruning
- **Overfitting:** Tree is too deep, memorizes noise in training data
- **Pre-pruning:** Stop growing early (e.g., $\chi^2$ test)
- **Post-pruning:** Grow fully then prune back
- **Cross-validation:** k-fold cross-validation to select model

### Attribute Selection
- **Information Gain:** Biased toward attributes with many values
- **Gain Ratio:** Corrects the above bias
- Handles both discrete and continuous attributes

---

## 4. Model Evaluation and Selection

### Learning Curves

Test set accuracy vs. training set size.

### Model Selection
- **Bias-Variance Tradeoff:** Simple model → high bias, complex model → high variance
- **Cross-validation:** Select the model with best generalization
- **Loss Function:** Different errors have different costs (e.g., false negative costs more in medical diagnosis)

### Regularization

Add a complexity penalty to the loss function to prevent overfitting:

$$\text{Loss}(w) = \sum_j (y_j - h_w(x_j))^2 + \lambda \|w\|^2$$

- **L1 Regularization:** Produces sparse models (many weights become 0)
- **L2 Regularization:** Weights shrink smoothly

---

## 5. Computational Learning Theory

### PAC Learning (Probably Approximately Correct)
- Goal: Find a hypothesis with error $\leq \epsilon$ with probability $1-\delta$
- **Sample complexity:** $N \geq \frac{1}{\epsilon}(\ln|H| + \ln\frac{1}{\delta})$
- Larger hypothesis space → more samples needed
- **VC Dimension** (Vapnik-Chervonenkis Dimension): Measures the expressive power of the hypothesis space

### PAC Learning of Decision Lists

$k$-DL (k-decision lists) can be efficiently PAC learned.

## 6. Linear Models

### Linear Regression

$$h_w(x) = w_0 + w_1 x_1 + \cdots + w_n x_n = w \cdot x$$

- **Loss function:** $L_2$ loss = $\sum_j (y_j - w \cdot x_j)^2$
- **Solution:** Gradient descent or normal equation $w = (X^TX)^{-1}X^Ty$

### Perceptron
- Hard threshold linear classifier: $h(x) = \text{sign}(w \cdot x)$
- **Perceptron learning rule:** $w_i \leftarrow w_i + \alpha(y - h(x)) \cdot x_i$
- Convergence guaranteed only when data is **linearly separable**

### Logistic Regression
- Uses sigmoid (soft threshold) instead of hard threshold: $g(z) = \frac{1}{1+e^{-z}}$
- Better for noisy and non-linearly separable data
- Optimized via gradient descent

## 7. Artificial Neural Networks

### Basic Structure
- **Neuron model:** $a_j = g(\sum_i w_{i,j} a_i)$
- **Activation Functions:**
  - Hard threshold → Perceptron
  - Sigmoid → Sigmoid Perceptron (differentiable)

### Network Types
- **Feed-forward Network:** DAG structure, no cycles, no internal state
- **Recurrent Network:** Has feedback connections, can support short-term memory

### Multi-Layer Feed-Forward Networks
- **Hidden Layers:** Layers between input and output
- **Universal Approximation:** Sufficient hidden units can represent any function

### Back-Propagation

Core algorithm for training multi-layer networks:

1. **Forward pass:** Input → hidden layer → output, compute output values
2. **Compute output error:** $\Delta_k = (y_k - a_k) \cdot g'(in_k)$
3. **Back-propagate error:** $\Delta_j = g'(in_j) \sum_k w_{j,k} \Delta_k$
4. **Update weights:** $w_{j,k} \leftarrow w_{j,k} + \alpha \cdot a_j \cdot \Delta_k$

### Practical Considerations
- Initialize weights to small random numbers
- Learning rate $\alpha$ selection: too small → slow convergence, too large → oscillation
- **Overfitting:** Larger networks are more prone to overfitting
- **Architecture selection:** Determining number of hidden layers and nodes

## 8. Non-Parametric Models

### k-Nearest Neighbors (kNN)
- At prediction time, find k nearest neighbors, vote/average
- **Pros:** Simple, no training required
- **Cons:** Slow prediction, must store all data, curse of dimensionality

### Efficient Lookup
- **k-d Trees:** Efficient lookup in low-dimensional spaces
- **Locality-Sensitive Hashing (LSH):** Approximate lookup in high dimensions

### Non-Parametric Regression
- **Locally Weighted Regression:** Weighted fitting of nearby points

---

## 9. Support Vector Machines (SVM)

### Core Idea
- **Maximum Margin Separator:** Choose the decision boundary farthest from all sample points
- **Support Vectors:** Sample points closest to the boundary

### Kernel Trick
- Maps data to higher-dimensional space, making non-linearly separable data linearly separable
- Common kernel functions: polynomial kernel, RBF (Gaussian) kernel
- No need to explicitly compute points in high-dimensional space

### Advantages
- Maximum margin → good generalization
- Kernel trick → complex nonlinear boundaries
- Non-parametric method, but practically retains only a few support vectors

---

## 10. Ensemble Learning

### Basic Idea

Combine predictions of multiple hypotheses to reduce individual errors.

### Boosting (AdaBoost)
1. Initial weights are equal
2. Train weak learner $h_k$
3. Increase weights of misclassified samples
4. Final hypothesis = weighted vote

**Key property:** Even with a weak learner (accuracy > 50%), boosting can achieve a perfect classifier.

### Decision Stumps

Single-level decision trees as weak learners.

### Online Learning
- Aggregate opinions of multiple experts
- Guaranteed performance close to the best expert (Hedge algorithm)

## 11. Knowledge in Learning (Ch 19)

### Role of Prior Knowledge
- Eliminates hypotheses inconsistent with prior knowledge
- "Fills in" explanations, allowing shorter hypotheses
- Faster learning from fewer samples

### Explanation-Based Learning (EBL)

Extract general rules from a single example through **explanation** and **generalization**:
1. Construct an explanation (proof tree) for the example
2. Generalize constants in the explanation

### Inductive Logic Programming (ILP)

Inductive learning in **first-order logic**:
- **Top-down:** Start from very general rules and refine
- **Bottom-up:** Reverse the deduction process
- Can automatically generate new predicates

### Relevance-Based Learning (RBL)

Use **determinations** (e.g., "Size determines Color") to identify relevant attributes, reducing the hypothesis space.

## 12. Learning Probabilistic Models (Ch 20)

### Bayesian Learning

**Core idea:** Perform inference over the posterior probability of the hypothesis space.

$$P(h_i | d) = \frac{P(d|h_i) P(h_i)}{\sum_j P(d|h_j) P(h_j)}$$

**Prediction:**

$$P(X | d) = \sum_i P(X | h_i) P(h_i | d)$$

- **Optimality:** Bayesian prediction is optimal given the prior
- **Cost:** Requires summation/integration over hypothesis space, usually infeasible

### MAP Hypothesis

Maximum a posteriori hypothesis: $h_{MAP} = \arg\max_{h_i} P(h_i | d)$

- Simpler than full Bayesian inference (optimization vs summation)
- Embodies **Ockham's Razor:** MAP = simplest hypothesis consistent with data

### MDL (Minimum Description Length)
- Minimize description length of hypothesis + description length of data given hypothesis
- Equivalent to MAP (logarithmic transformation)

### Maximum Likelihood (ML)

Assume uniform prior: $h_{ML} = \arg\max_{h_i} P(d | h_i)$

---

## 13. Parameter Learning with Complete Data

### Maximum Likelihood Parameter Learning

**Steps:**
1. Write the likelihood function $L(\theta) = \log P(d | \theta)$
2. Differentiate with respect to parameters
3. Set derivative to zero, solve

**Example:** Candy problem
- $\theta$ = proportion of cherry
- Observe $c$ cherry, $\ell$ lime
- $\theta_{ML} = \frac{c}{N}$ (observed proportion)

### Naive Bayes
- Assumes attributes are **conditionally independent** given the class
- Requires few parameters, data-efficient
- Performs well in many practical tasks

### ML Interpretation of Linear Regression

Maximizing likelihood = minimizing $L_2$ loss (assuming Gaussian noise).

### Bayesian Parameter Learning
- Define parameter prior $P(\theta)$
- Update posterior after data arrives
- Superior to ML with small datasets

|---|---|---|
| Decision Trees | Supervised | Information gain, interpretable |
